# Creative Portfolio – Project Instructions

Build a **premium, creative, production-ready portfolio website** for a freelance developer. This should **not** resemble a typical developer portfolio with the standard Hero → About → Skills → Projects → Contact flow.

The portfolio must feel like a **high-end product website** that demonstrates design thinking, engineering quality, and attention to detail. Every interaction should reinforce professionalism and creativity.

The final experience should be memorable enough that potential clients immediately recognize the developer as someone who delivers premium work.

# Important Rules

DO NOT create a generic developer portfolio.

DO NOT use common portfolio templates.

DO NOT create sections just because every portfolio has them.

Think like a Creative Director designing a premium digital product.

The website should feel like a premium experience from the first interaction until the footer.

Every design decision should have a purpose.

Every animation should add value.

Every component should be reusable.

Every layout should feel intentional.

The final codebase must be production-ready, maintainable, scalable, performant, and follow senior-level engineering practices with strict TypeScript, clean architecture, reusable components, and consistent coding conventions.
